Princípios de design
01
Sem LLM no roteamento
O Capability Graph é determinístico — Java puro. O LLM só entra na extracção e síntese. Matching de skills nunca falha por alucinação.
02
Contrato imutável
AgentOutput com confidence é o contrato de todos os nós. O hub nunca sabe o que está dentro de um nó — só lê o contrato.
03
Resultado parcial sempre avança
Timeout de 8s por nó. Se um nó não responde, a quest avança sem ele. O sistema nunca bloqueia por causa de um nó lento.
04
Zero cloud por defeito
Tudo corre local com Ollama. phi3:mini em 2GB de RAM. Dados nunca saem da máquina. Custo zero para a guilda inteira.
05
Cada dev é o owner
Cada nó é um processo independente — porta própria, modelo próprio. Pode ser desligado sem afectar os outros. Autonomia total.
06
Crescimento orgânico
O hub não é omnisciente. Resolve só o que as skills disponíveis permitem. Cada novo herói expande o que a guilda consegue fazer.
